## ADDED Requirements

### Requirement: CLI reference documents all subcommands and flags
`docs/reference/cli.md` SHALL document every `cornet` CLI subcommand (`run`, `view`, `ui`) with: description, positional arguments, optional flags, and one example invocation each.

#### Scenario: CLI reference covers the ui subcommand
- **WHEN** a user reads `docs/reference/cli.md`
- **THEN** they SHALL find the `cornet ui` subcommand documented with its `--port` flag and example output

### Requirement: Leaderboard format reference documents the JSON schema
`docs/reference/leaderboard-format.md` SHALL describe the structure of `leaderboard.json`: top-level array, per-entry fields (timestamp, variant_id, metric value, status), and the atomic-write guarantee.

#### Scenario: Format reference explains atomic write
- **WHEN** a developer reads `docs/reference/leaderboard-format.md`
- **THEN** they SHALL find an explanation of the tmp-file-then-rename pattern used by `leaderboard/writer.py`

### Requirement: Config schema reference is auto-generated and marked as such
`docs/reference/config-schema.md` SHALL contain a prominent header note: "This file is auto-generated by `scripts/gen_schema_docs.py`. Do not edit manually." It SHALL be re-generated by `make docs` and checked for freshness by `make docs-check`.

#### Scenario: Manual edits are overwritten on next generate
- **WHEN** a developer manually edits `docs/reference/config-schema.md` and then runs `make docs`
- **THEN** the manual edits SHALL be overwritten by the generator output
